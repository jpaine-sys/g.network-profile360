Context

How Flex Credits work today
and where the limitations are

Under a pure Flex Credit model, every core Data 360 operation draws from the same pool. For teams doing regular data work, this creates a few practical issues worth addressing.

Usage uncertainty
Credit consumption is hard to predict in advance
How many credits a segment refresh or identity resolution run uses depends on data volumes and frequency — both of which change. That makes forward planning difficult.
Email activity triggers unification
Every email open or click can re-trigger identity resolution
When email engagement events flow back into Data 360 in real time, each one can kick off a new unification run. On active campaigns this adds up quickly and is difficult to anticipate.
Core operations are the main cost driver
90–95% of typical base consumption comes from four meters
Salesforce data shows that Prep, Unification, Segmentation and Activation account for the vast majority of Flex Credit usage for standard CDP workflows — before any AI use cases are factored in.
The Profile SKU is designed specifically to address this: it removes metering from those four operations entirely, leaving Flex Credits available for Agentforce and other AI-driven use cases.

Profile SKU · Included Operations

What the Profile SKU
includes at no additional cost

The Profile SKU removes metering from the four operations that make up the majority of standard Data 360 usage. None of these draw from your Flex Credits.

Identity Resolution & Unification Stitches every data source into a single deduplicated profile per customer. Runs automatically whenever new data arrives — no credits consumed, no matter how often it fires.
Unmetered
Data Prep & Ingestion Brings in data from CRM, web, app, email engagement, and third-party sources. Cleans and harmonises it. Fully included — batch ingestion is free across every source.
Unmetered
Segmentation Build and refresh audience segments as frequently as your campaigns demand. Real-time and batch both included. Run it ten times a day if you need to.
Unmetered
Activation Push segments to marketing journeys, advertising platforms, and any channel. Every push, every time — no per-activation charge, no throttling.
Unmetered
Profile SKU · Further Detail

Additional capabilities
included with the Profile SKU

Beyond the four unmetered operations, the Profile SKU includes several capabilities that previously required separate add-ons or were not available at all.

⚡
Real-time profile access
Sub-second profile API access is included
This previously required a separate "Sub-Second Real-Time Profile & Entities" SKU. Under the Profile SKU, sub-second real-time profile access is included — no additional licence or cost required.
∞
Identity Resolution rulesets
No limit on the number of rulesets
The Profile SKU carries no cap on Identity Resolution rulesets. You can build as many match rules as your data setup requires. This restriction was removed across both standard and enterprise tiers in April 2026.
🧩
Anonymous profile weighting
Anonymous profiles count at 4% of a known profile
Anonymous records — unknown web visitors, unresolved contacts — are now weighted at 0.04 rather than 0.20. This change, made in 2026, means high-traffic datasets with a large proportion of anonymous records are sized more accurately.
🔒
Email-triggered re-unification
Email engagement events do not consume credits
Under Flex Credits, email opens and clicks flowing back into Data 360 can re-trigger identity resolution runs, adding to credit consumption unexpectedly. Under the Profile SKU, those runs are unmetered — email activity does not affect your credit balance.
